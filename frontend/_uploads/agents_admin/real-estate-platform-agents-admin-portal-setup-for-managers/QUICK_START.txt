❓ COMMON QUESTIONS
─────────────────────────────────────────────────────────────────────

Q: How do I add a new admin manager?
A: Currently hardcoded. In production, add to database and verify
   against backend API. See AdminAuthContext.jsx

Q: Why are the changes not saved after refresh?
A: Mock data is stored in React state. Add backend API to persist.
   See ADMIN_PORTAL_GUIDE.md for production checklist.

Q: Can I change the demo password?
A: Yes, edit AdminAuthContext.jsx lines 36-37. But this is temporary!

Q: How do I make this production-ready?
A: Follow the checklist in ADMIN_PORTAL_GUIDE.md section "Security &
   Authentication". Key: backend API + JWT + password hashing.

Q: Where's the admin link in the navbar?
A: Intentionally removed! Users can only access via direct URL or
   bookmark. This is for security (obscurity, not security measure).
